# ContentFoundry

> ContentFoundry is a headless CMS with an integrated knowledge base and AI writer. Built for founder-led brands and small teams who need to publish at scale without losing their voice or context.

Full reference: https://getcontentfoundry.com/cf-llms-full.txt

## What it is

ContentFoundry is a content supply chain platform — not just a CMS, not just an AI writer. It connects what a brand knows to what it publishes. The knowledge base (product briefs, competitor research, style guides, research documents) feeds the AI writer, which feeds the CMS, which exposes content via a REST API and TypeScript SDK.

Built in production. Powers NutraGLP (nutraglp.com), a supplement brand running entirely on this stack.

## Who it's for

- Founder-operators with a content acquisition thesis who are currently duct-taping Notion + CMS + ChatGPT
- Marketers who have deep institutional knowledge that doesn't connect to their content output
- Agency owners managing multiple client content operations

## Core capabilities

- Knowledge Base: Upload PDFs, DOCX, TXT, MD, CSV — documents are parsed and made available to AI at generation time
- AI Writer: Brand-aware drafts with knowledge base context injection
- Blog CMS: Scheduled publishing, tags, metadata, versioning, audit log
- Pages CMS: Dynamic pages with full content control via API
- Products: Product catalog with variants, pricing, rich copy
- REST API: Full REST API with TypeScript SDK
- Webhooks: Dispatch on publish, update, delete
- Role-based Access: Admin, editor, viewer — enforced at the API level
- Content Versioning: Snapshot on every save, restore any version
- BYOK Integrations: Unsplash, SendGrid, Cloudinary, ElevenLabs, remove.bg

## Technical

- Built on Next.js
- Deployable to Netlify or Vercel
- SQLite-backed via Turso (LibSQL)
- REST API with OpenAPI spec
- TypeScript SDK: @contentfoundry/sdk
- BYOK (Bring Your Own Key) model for all third-party integrations — keys stored in site settings, pulled at runtime
